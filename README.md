# Veritas AI: Universal Learning Pain-point Diagnostic Engine

> **"학습의 완성은 정답을 아는 것이 아니라, 어디서 사고가 멈추는지 스스로 발견하는 데서 시작됩니다."**
>
> Veritas AI는 사용자의 다양한 입력(개념, 코드, 언어, 문제 상황, 실수 패턴)을 분석하여 **지식의 공백(Pain-point)과 사고 병목 지점**을 정밀하게 진단하는 AI 학습 보조 솔루션입니다.

---

## 1. 기획 의도 (Project Intent)
이 프로젝트는 실제로 공부하면서 느낀 한 가지 문제에서 시작되었습니다. 분명 개념 설명을 보면 이해한 것 같지만, 막상 문제나 새로운 상황에 적용하면 같은 실수를 반복하는 경험이 자주 있었습니다.

기존 AI는 정답을 빠르게 제시해 주지만, 사용자가 **왜 그 지점에서 반복적으로 막히는지** 까지는 충분히 보여주지 못한다고 느꼈습니다. 그래서 단순 답변보다 **사고가 멈춘 지점을 먼저 찾는 진단형 AI**를 만들고자 했습니다.

기존의 학습형 AI는 대부분 정답이나 설명을 즉시 제공하는 데 집중되어 있습니다. 하지만 실제 학습 실패는 단순한 정보 부족이 아니라, **이해 → 구조화 → 적용 → 비교 → 예방** 중 특정 사고 단계에서 발생하는 경우가 많습니다.

본 프로젝트는 사용자가 입력한 질문, 코드, 문장, 오류, 실수 경험을 바탕으로 **역진단(Reverse-Diagnosis)** 방식의 맞춤형 Yes/No 질문을 생성하여, 학습자가 자신의 사고 흐름이 어느 지점에서 끊기는지 명확히 인지하도록 돕는 데 초점을 맞췄습니다.

---

## 2. 주요 기능 (Key Features)
- **광범위 입력 기반 지식 진단**: 특정 과목에 한정되지 않고 코드, 외국어 문장, 개념 질문, 문제 상황, 실무 고민 등 다양한 입력을 분석합니다.
- **인지 결함 역추적 (Cognitive Backtracking)**: 사용자의 No 응답 패턴을 기반으로 어떤 사고 단계에서 논리가 무너졌는지 추적합니다.
- **정밀 진단 리포트 생성**:
  - 결손 개념 및 병목 단계 탐지
  - 반복 실수 패턴 분석
  - 즉시 복습해야 할 우선 개념 제안
  - 재발 방지용 개인화 학습 방향 제시

## 4. 시스템 로직 (Logic Flow)
1. **Input**: 사용자가 개념, 코드, 언어 문장, 오류, 문제 상황 등을 자유롭게 입력
2. **Inference**: 입력 타입과 숨겨진 실수 신호(signal) 추론
3. **Diagnosis**: 서로 다른 사고 단계를 겨냥한 맞춤형 Yes/No 질문 5개 생성
4. **Detection**: No 응답 기반으로 결손 개념 및 반복 실수 패턴 추적
5. **Report**: 최종 Root-Cause 진단 결과 및 복습 로드맵 생성

## 6. Innovation Point
대회 관점에서 이 프로젝트의 핵심은 단순한 LLM 활용이 아니라, **질문 설계 로직과 학습 실패 구조화 방식**에 있습니다.

특히 다음 3가지를 차별점으로 두었습니다.
- 입력을 그대로 반복하지 않고 숨겨진 실수 신호(signal)를 추론
- 질문 5개를 모두 서로 다른 사고 단계로 분리
- Yes/No 응답을 통해 빠르게 병목 구간을 수집

이 부분은 직접 여러 입력 케이스를 테스트하면서 반복적으로 수정한 핵심 설계 포인트입니다.

이 프로젝트의 핵심 차별점은 AI의 역할을 단순한 **정답 생성기**가 아니라,

> **학습 실패 원인 진단 엔진**

으로 재정의했다는 점입니다.

즉 사용자가 무엇을 모르는지가 아니라,

> **왜 같은 실수를 반복하는지**

를 추적하는 데 초점을 둡니다.

이 구조는 교육뿐 아니라:
- 코딩 디버깅
- 언어 학습
- 실무 훈련
- 창의적 문제 해결

영역까지 확장 가능한 범용 인지 진단 플랫폼으로 발전할 수 있습니다.
